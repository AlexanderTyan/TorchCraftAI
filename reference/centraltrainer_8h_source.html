40 /**
41  * A trainer that sends episodes to one or more central instances.
42  *
43  * In this trainer, several "server" instances will collect episode data from
44  * "client" instances. Users are required to subclass this and override
45  * `receivedEpisode()`, which will be called on server instances whenever a new
46  * episode arrives. The trainer can be used like any other trainer, but ideally
47  * there should be no calls to sleep() between `update()` calls to ensure fast
48  * processing of collected episode data.
49  *
50  * Implementation details:
51  * The trainer spawns dedicated threads for servers and clients.
52  * The data that goes over the network (serialized episodes) will be compressed
53  * using Zstandard, so there's no need to use compression to your custom replay
54  * buffer frame structure.
55  * Episode (de)serialization (including (de)compression) will be performed in
56  * the respective thread calling `stepEpisode()` (client) and `update()`
57  * (server).
58  *
59  * TODO: Extend this so that it can be used in RL settings.
60  */
