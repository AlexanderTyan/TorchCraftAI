80 /**
81  * Helper class for job coordination via a central Redis instance.
82  *
83  * In a nutshell, the Cpid2kWorker class does the following:
84  * - Communicate local job status to the scheduler via Cpid2kHeartbeater (ctor
85  * will throw if job does not have start permission).
86  * - Provide basic information about global job status (`peers()`, `isDone()`,
87  * etc.) and local status as seen by the scheduler (`consideredDead()`).
88  * - Convenience functions for common operations (`dcontext()`,
89  * `waitForOne/All()`, etc.)
90  *
91  * For manual operations on the Redis database, use `threadLocalClient()` to
92  * obtain a RedisClient instance for the current thread. Note that these will be
93  * re-used.
94  *
95  * All public functions are thread-safe, i.e. it's alright to call them from
96  * several trainer or game threads.
