65 /**
66  * Client for ZeroMQ REQ-REP pattern.
67  *
68  * This client can be connected to multiple ReqRepServers and will send out
69  * requests in a round-robin fashion. It provides some basic robustness
70  * regarding slow or crashing servers by implementing the ZeroMQ Lazy Pirate
71  * pattern: if the server does not send a reply in time, retries will be
72  * attempted. Once a reply has been received, the `handleReply()` function will
73  * be called (to be provided by subclasses). The server list can be updated
74  * without loss of messages.
75  *
76  * The client keeps a backlog of requests that could not be delivered yet and
77  * for which delivery should be attempted at a later point. The size of this
78  * backlog has to be specified during construction.
79  *
80  * A few words regarding retries and the callback function: since this class
81  * provides a purely synchronous interface, send and receive operations are only
82  * performed during `request()`, `waitForReplies()` and `updateEndpoints()`
83  * calls. These are the functions that might result in a call to the supplied
84  * callback function.
85  */
